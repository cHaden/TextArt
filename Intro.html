Word Art Generator:

Main project
Local copy

We print a lot of word art, so I wanted to make a basic app that could take a single word or phrase and generate a useable image file.

We start with the default settings, and can select a word or phrase to render
...a color palette to use (it randomly assigns colors from the selected palette)
...a font set to use (it also randomly assigns fonts from the selected set. These are pulled down from Google fonts)
...a layout to use (Log Cabin or Straight. Straight wraps the words horizontally so the image tiles)
...a background texture if desired
...total number of repeats to display (fit to the canvas size)
...and an option for the text to get larger, smaller, or stay the same from the center out (not applicable for straight layout)

Buttons:
-- update: redraw with the selected options
-- Export as PNG: save canvas as a raster file
-- Export as SVG: save canvas as a vector file (in development)

Obvious Problems:

-- I am not a graphic designer. The code does the Thing, but the Thing doesn't look completely great
-- SVG output is broken! There is no easy fix for this. Could rasterize all of the text, but that completely removes any benefit to exporting in a vector format
Example
-- Short names don't look great. Ex. "Jo", "Dan"
-- Long names are *okay* Ex. "Ozymandias" Very long phrases look bad. Ex. "This is a super long phrase and looks silly"
-- Fabric.js isn't completely accurate in calculating the bounding box for text-- Some text gets clipped, expecially with italics and script. Could stick to "safe" fonts, or add more whitespace to buffer
-- File resolution is not configurable. A 600x600 file is about 4" at 150 DPI, but it's likely that people might want larger prints
